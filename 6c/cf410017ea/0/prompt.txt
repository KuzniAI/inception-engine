There are two issues with paths in manifest
1. Manifest-controlled skill.name can escape the agent skill directory and turn deploy/revert into arbitrary file deletion under the invoking user. src/config/manifest.ts (line 47), src/core/resolve.ts (line 51), src/core/deploy.ts (line 70), src/core/revert.ts (line 55)
skill.name is only checked for non-empty string, then interpolated directly into the target path template. A manifest entry like "name": "../../.ssh" or one containing platform separators can resolve outside ~/.codex/skills, ~/.claude/skills, etc. On deploy, removeExisting() will recursively delete whatever already exists at that escaped target; on revert, rmSync() will remove it again. This is the most serious security issue in the repo.
2. Manifest-controlled skill.path can point outside the provided repository and expose arbitrary local directories to AI agents. src/config/manifest.ts (line 51), src/core/deploy.ts (line 25)
planDeploy() uses path.resolve(sourceDir, skill.path) but never verifies that the resolved path stays inside sourceDir. A malicious or mistaken manifest can use ../.. to copy or symlink sensitive local paths into an agent-readable skill location. Because the tool is explicitly meant to deploy from third-party repos, this needs a containment check against the manifest root.
